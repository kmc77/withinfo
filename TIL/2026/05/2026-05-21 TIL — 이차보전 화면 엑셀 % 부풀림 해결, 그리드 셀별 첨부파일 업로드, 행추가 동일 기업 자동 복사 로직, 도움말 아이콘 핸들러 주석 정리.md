## 1. 엑셀 다운로드 % 컬럼 부풀림 — `excelCellType="text"`로 해결

### 증상

같은 그리드 안 % 컬럼 2개 중 한쪽만 엑셀 다운로드 시 ×100 부풀림.

| 컬럼 | 의미 | DB raw | 화면 표시 | 엑셀 표시 |
|---|---|---|---|---|
| `assrRat` | 보증요율 | 0.013, 0.015 (소수) | 0.01% | 정상 |
| `intRto` | 이차보전율 | 9.96, 6, 2 (정수대) | 9.96% | **996.00% (×100 부풀림)** |

### 원인

- 화면 `displayFormat="##.00%"`은 raw 값에 `%` 기호만 붙임 (×100 없음)
- 엑셀 다운로드 시 WebSquare가 셀에 **백분율 서식**을 박음
- 엑셀 백분율 서식 = raw × 100해서 표시
- `assrRat`은 raw가 소수(0.013)라 ×100 돼도 1.30%로 우연히 자연스럽게 나옴
- `intRto`은 raw가 정수대(9.96)라 ×100 시 996.00%로 폭발

### 해결

```xml
<w2:column id="intRto" excelCellType="text" ... />
```

`excelCellType="text"` 속성으로 엑셀 셀 타입을 텍스트 강제 → 엑셀의 백분율 서식 해석 차단 → 화면 표시값 그대로 셀에 박힘.

### 시도와 결과 정리

| 시도 | 결과 |
|---|---|
| `type:1` + `useDataFormat:false` | `intRto`만 부풀림 |
| `type:0` + `useDataFormat:true` | `%` 기호 사라지고 숫자만 |
| `textAlign="right"` 제거 | 동일 부풀림 |
| `dataType` number ↔ bigDecimal | 동일 |
| **`excelCellType="text"` 추가** | **해결** |

### 교훈

- `displayFormat`은 화면 렌더링용. 엑셀 다운로드는 셀 타입/서식 별개 경로
- 같은 displayFormat인데 한 컬럼만 깨질 때 = **DB raw 스케일 불일치 + 엑셀 자동 서식 해석** 콤보 의심

## 2. 그리드 클릭 이벤트 분기 버그 — OR/AND 혼동

### 잘못된 코드

```javascript
if(this.getColumnIndex("rmmdFileGrpId") != col) return;
if(col != this.getColumnIndex("sprtTrgtEvdcPprsFileGrpId")) return;
if(col != this.getColumnIndex("loanNtshFileGrpId")) return;
```

3개 컬럼 중 하나라도 일치하면 통과시키려는 의도(OR)였으나, 순차 if + return은 **AND처럼 동작**. col은 한 번에 하나의 값이라 3개 컬럼 모두에 동시 매칭 불가 → 무조건 어딘가에서 return.

### 의도 반영 수정

```javascript
if(col != this.getColumnIndex("rmmdFileGrpId") 
&& col != this.getColumnIndex("sprtTrgtEvdcPprsFileGrpId") 
&& col != this.getColumnIndex("loanNtshFileGrpId")) {
    return;
}
```

### 콘솔 디버깅 함정

```javascript
console.log("col:", col);   // ← if 위에 있어서 무조건 찍힘
if(...) return;             // ← 실제 return 타도 위 로그는 항상 보임
```

콘솔 로그가 if문 위에 있어서 "값이 정확히 찍히니까 통과하는 거"로 오판 → 분기 결과 진단 실패.

**개선:** if문 양쪽에 로그 박기.

```javascript
if(...) {
    console.log("→ RETURN");
    return;
}
console.log("→ PASS");
```

## 3. `getColumnIndex` 두 번째 인자 함정 (재확인)


`getColumnIndex`는 컬럼명만 받음. 두 번째 인자 `col`을 fallback으로 해석하거나 무시 → `col == col` → 항상 false → return 안 탐.

```javascript
if(col != grd_list.getColumnIndex("inrtCmptYmd")) return;  // ✓
```

## 4. 그리드 셀별 첨부파일 업로드 구현 (SFPS_CSBS018M)

### 요구사항

그리드 행의 4개 파일그룹ID 컬럼 셀 클릭 시 업로드 팝업 호출 → 업로드 완료 후 짝이 되는 Nm 컬럼에 원본 파일명 바인딩.

| FileGrpId 컬럼 | Nm 컬럼 |
|---|---|
| `acinRpppFileGrpId` | `acinRpppFileNm` |
| `rmmdFileGrpId` | `rmmdFileNm` |
| `sprtTrgtEvdcPprsFileGrpId` | `sprtTrgtEvdcPprsFileNm` |
| `loanNtshFileGrpId` | `loanNtshFileNm` |

### 구현

```javascript
/**
 * 첨부파일 등록 - 그리드 셀 클릭 진입점
 */
scwin.btn_addFile_onclick = function(row, col) {
    var idxAcin = this.getColumnIndex("acinRpppFileGrpId");
    var idxRmmd = this.getColumnIndex("rmmdFileGrpId");
    var idxSprt = this.getColumnIndex("sprtTrgtEvdcPprsFileGrpId");
    var idxLoan = this.getColumnIndex("loanNtshFileGrpId");
    
    var colId, nmColId;
    if(col === idxAcin)      { colId = "acinRpppFileGrpId";          nmColId = "acinRpppFileNm"; }
    else if(col === idxRmmd) { colId = "rmmdFileGrpId";              nmColId = "rmmdFileNm"; }
    else if(col === idxSprt) { colId = "sprtTrgtEvdcPprsFileGrpId";  nmColId = "sprtTrgtEvdcPprsFileNm"; }
    else if(col === idxLoan) { colId = "loanNtshFileGrpId";          nmColId = "loanNtshFileNm"; }
    else return;
    
    // 콜백 컨텍스트 저장 (문자열 콜백명이라 클로저 불가)
    scwin._uploadRow     = row;
    scwin._uploadColId   = colId;
    scwin._uploadNmColId = nmColId;
    
    var fileGrpId = dlt_loanIntSprtBizListVO.getCellData(row, colId);
    
    com.win.fn_fileUplodPop( fileGrpId
                          , 50
                          , ''
                          , ''
                          , 'scwin.callbackUploadDone');
};

/**
 * 첨부파일 등록 콜백 - 업로드 결과를 그리드 셀에 바인딩
 */
scwin.callbackUploadDone = function(e) {
    var jsonD = e.retJson;
    if(!jsonD || !jsonD.length) return;
    
    var fileInfo = jsonD[0];
    var dl       = dlt_loanIntSprtBizListVO;
    var row      = scwin._uploadRow;
    
    dl.setCellData(row, scwin._uploadColId,   fileInfo.fileGrpId);
    dl.setCellData(row, scwin._uploadNmColId, fileInfo.fileOtxtNm);
};
```

### 핵심 패턴

**A. 콜백 컨텍스트 보관 (`scwin._xxx`)**

`fn_fileUplodPop`이 콜백을 **문자열 이름**(`'scwin.callbackUploadDone'`)으로 받음 → 클로저 사용 불가 → row/colId 등 컨텍스트를 scwin 전역에 임시 저장 필요. `_` 접두사 = 내부용 임시 변수 컨벤션.

**B. `e.retJson` 구조 (배열)**

| 키 | 값 예시 | 비고 |
|---|---|---|
| `fileGrpId` | `""` | **업로드 직후 빈값** (서버 저장 시 채번 추정) |
| `fileNm` | `"20260521_3c4c4c46-..."` | 서버 저장용 UUID 이름 |
| `fileOtxtNm` | `"스크린샷 2026-04-07.png"` | **원본명 (화면 표시용)** |
| `fileExtnNm` | `"png"` | |
| `fileSz` | `116730` | |
| `rowStatus` | `"C"` | |

**→ 화면 표시는 `fileOtxtNm`**, `fileNm`은 UUID라 사용자에게 안 보임.

## 6. 도움말 아이콘 헤더 핸들러 주석 정리

기존 코드의 의도를 추후 유지보수자가 빠르게 파악할 수 있도록 주석 보강.

### 헤더 클릭 시 팝업 토글

```javascript
/**
 * 그리드 헤더의 도움말("?") 아이콘 클릭 시 팝업 토글
 * - column49: 도움말 아이콘 전용 컬럼
 * - udc_help: 별도 absolute 배치된 팝업 컴포넌트
 */
scwin.grd_loanIntSprtBizListVO_onheaderclick = function(headerId) {
    if(headerId === "column49"){
        var btnColumn = this.dom.head_tds[9];           // 아이콘 컬럼
        var offsetLeft = btnColumn.offsetLeft - 30;     // 중앙 정렬 보정
        udc_help.setStyle("left", offsetLeft + "px");
        udc_help.childControlList.forEach(function(child){
            child.toggleClass("on");                    // 팝업 표시/숨김 토글
        })
    }
};
```

### 좌우 스크롤 시 위치 동기화 + 자동 숨김

```javascript
/**
 * 좌우 스크롤 시 도움말 아이콘/팝업 위치 동기화 및 자동 숨김
 * - 대상 컬럼이 frozen 영역에 가려지면(25px 이하) 아이콘+팝업 숨김
 */
scwin.grd_loanIntSprtBizListVO_onscrollx = function() {
    var helpTargetColumn = this.dom.head_tds[8];   // 도움말 대상(선사분류)
    var btnColumn        = this.dom.head_tds[9];   // 아이콘 컬럼
    var offsetLeft       = btnColumn.offsetLeft - 30;
    udc_help.setStyle("left", offsetLeft + "px");
    
    if(helpTargetColumn.offsetWidth <= 25){
        btnColumn.classList.add("hideAfter");      // 아이콘 숨김
        udc_help.childControlList.forEach(function(child){
            child.removeClass("on");               // 팝업도 닫음
        })
    }else{
        btnColumn.classList.remove("hideAfter");
    }
};
```

## 7. 부수적으로 정리한 WebSquare/CSS 지식

### A. `rowStatusVisible` — 그리드 행 상태 아이콘 표시

```xml
<w2:gridView 
    rowStatusVisible="true"
    rowStatusWidth="40"
    rowStatusHeaderValue="상태"
    ...>
```

기본값 false. true로 두면 각 row의 상태가 별도 컬럼에 아이콘으로 표시됨.

**rowStatus 값:**
- `C` (Create) — addRow로 신규 추가
- `U` (Update) — 셀 수정
- `D` (Delete) — removeRow로 삭제 (`saveRemovedData="true"`일 때만 보존)
- `R` (Read) — 변경 없음

**저장 시 변경분만 추출:**

```javascript
var modified = dlt_list.getModifiedJSON();  // C/U/D만
```

### B. CSS 강제 적용 — `!important`

```css
.col_2 { width: 20% !important; }
```

**우선순위 (강한 순):**
1. `!important`
2. 인라인 `style="..."`
3. ID selector → class selector → tag selector

남발 금지. `!important`끼리 충돌하면 다시 우선순위 따짐.

### C. GitHub Desktop 충돌 해소

1. 충돌 파일 열기 → `<<<<<<<`, `=======`, `>>>>>>>` 마커 직접 편집
2. 셋 다 제거 후 저장
3. "Continue merge" 클릭
4. **본인 변경분으로 강제 푸시**: `git push origin kmc77 --force-with-lease`
   - `--force`보다 안전 (다른 사람 push 있으면 막아줌)
   - **개인 브랜치(`kmc77`)만 사용. 공용 브랜치 금지**
