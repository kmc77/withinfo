# 2026-05-12 이차보전 금액 검증 — JDBC-5074 디버깅 착수

## 작업 배경

- **화면:** 이차보전 금액 검증
- **매퍼:** `IdstAmtCmptSql.xml` / `selectIdstAmtCmptList`
- **패키지:** `kobc.ngs.sf.ps.csbs`
- **증상:** 특정 달(202401, 202403 등)에만 `JDBC-5074: Given string does not represent a number in proper format` 발생

---

## 트러블슈팅 과정

### 1. 컬럼명 오류 발견 — JDBC-8026

- **증상:** `Invalid Identifier` — `B.RPMT_YMD` 참조 불가
- **원인:** B 테이블(`TGU_GRNF_CMPT_I`)에 `RPMT_YMD` 컬럼 없음. 실제 컬럼명은 `EARL_RPMT_YMD`
- **해결:** SELECT절 `B.RPMT_YMD` → `B.EARL_RPMT_YMD` 교체

### 2. WHERE절 타입 불일치

- **증상:** VARCHAR 날짜 컬럼을 DATE와 직접 비교 → Tibero 암묵 변환 실패
- **AS-IS:**
```sql
AND B.RPMT_YMD >= TRUNC(TO_DATE('202601','YYYYMM'))
AND B.RPMT_YMD <= LAST_DAY(TO_DATE('202601','YYYYMM'))
```
- **TO-BE:**
```sql
AND B.RPMT_YMD >= TO_CHAR(TRUNC(TO_DATE('202601','YYYYMM')),'YYYYMMDD')
AND B.RPMT_YMD <= TO_CHAR(LAST_DAY(TO_DATE('202601','YYYYMM')),'YYYYMMDD')
```
- **원칙:** VARCHAR 날짜 컬럼 비교 시 DATE → `TO_CHAR(...,'YYYYMMDD')`로 타입 맞추기

### 3. 날짜 연산 빼기 문제

- **원인:** FST_CNT/SEC_CNT/TRD_CNT 계산식에서 VARCHAR끼리 `-` 연산 → Tibero 암묵 `TO_NUMBER` 시도 → 특정 데이터 조합에서 실패
- **해결 방향:**
```sql
-- AS-IS (위험)
F.RPMT_YMD - TO_CHAR(TRUNC(TO_DATE('202601','YYYYMM')),'YYYYMMDD') + 1

-- TO-BE (안전)
TO_DATE(F.RPMT_YMD,'YYYYMMDD') - TRUNC(TO_DATE('202601','YYYYMM')) + 1
TO_DATE(G.RPMT_YMD,'YYYYMMDD') - TO_DATE(F.RPMT_YMD,'YYYYMMDD')
LAST_DAY(TO_DATE('202601','YYYYMM')) - TO_DATE(G.RPMT_YMD,'YYYYMMDD')
```
- **상태:** 수정 후에도 에러 지속 — 쿼리 내 `-` 연산 잔여분 추가 탐색 필요

## 테이블 구조 파악

| 별칭 | 테이블명 | 비고 |
|------|----------|------|
| A | TGU_ASSR_APLY_I | 보증신청정보 |
| B | TGU_GRNF_CMPT_I | 보증로산출정보 |
| C | TGU_ASSR_APLY_I | 보증신정정보 |
| E/F/G | TGU_FNNC_CTRT_RPMT_SCHD_I | 금융계약상환일정정보 |

- E 테이블 `RPMT_YMD`: VARCHAR(8), PK 포함 — 정상
- B 테이블: `RPMT_YMD` 없음, `EARL_RPMT_YMD` VARCHAR(8) 존재

## 핵심 학습

### Tibero VARCHAR 날짜 연산 원칙
- VARCHAR 컬럼끼리 `-` 연산 절대 금지 → 반드시 `TO_DATE` 명시
- VARCHAR vs DATE 직접 비교 금지 → DATE를 `TO_CHAR`로 변환해서 맞추기
- JDBC-5074 = String→Number 암묵 변환 실패. 스택트레이스 `executeInternal`에서 터지면 쿼리 실행 단계(VO 매핑 이전)

### 디버깅 원칙
- 데이터 검증 쿼리 0건 나오면 가설 빨리 버리기
- 단계적 변환(한 개씩 수정 후 조회) → 원인 특정
- 한꺼번에 바꾸면 통과해도 진짜 원인 미궁
