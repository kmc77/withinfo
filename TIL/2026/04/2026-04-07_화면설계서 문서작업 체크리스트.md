## 1. 화면정의 시트 — 필드 길이 기준

| 접미사 | 타입 | 길이 |
|---|---|---|
| `~~Nm` (명) | String | 100 |
| `~~Cd` (코드) | String | 8 |
| `~~Ymd` (일자) | String | 8 |
| `~~Amt` (금액) | BigDecimal | 19 (19,0) |
| 이율 | BigDecimal | 12 (12,6) |

> 금액 필드는 포맷 `#,###.####` 확인 → BigDecimal / 길이 19

---

## 2. 화면처리흐름 — 자주 누락되는 항목

### 버튼 상태 처리 (`setBtnDisableById`)
- 화면오픈시: 어떤 버튼을 `disable(true)` 하는지 대상 명시 필요
- 신규버튼 클릭: `disable(false)` enable 처리 흐름 누락 잦음
- 수정 화면 복귀 후: 버튼 상태 복원 흐름도 기재 필요

```javascript
// 사용법
com.util.setBtnDisableById(true, "btn_reg,btn_upd");   // disable
com.util.setBtnDisableById(false, "btn_reg");           // enable
```

### 넘버링 체크
- 섹션 번호 중복 주의 (예: 5번이 두 개 → 두 번째를 6번으로 수정)
- 섹션 내부 세부 스텝: `1 → 1-1 → 2` 순서 확인
  - validation 체크 후 `1-1: 필수입력항목 미입력 시 오류처리` 누락 잦음

### 조회 섹션 패턴 (정형)
```
1. 필수입력항목 체크
1-1. 필수입력항목 미입력 시 오류 메시지 처리 (MsgID: EZZ0068)
2. 목록 조회 API 호출
```

## 3. DataList 누락 체크

- Outline > Head > DataCollection 목록과 화면처리흐름의 데이터 흐름을 비교
- DataList가 있는데 처리흐름에 바인딩/초기화 언급 없으면 누락 의심
- 예: `dlt_hddpList` 처럼 팝업 연동용 hidden 리스트도 처리흐름에 기재 필요
