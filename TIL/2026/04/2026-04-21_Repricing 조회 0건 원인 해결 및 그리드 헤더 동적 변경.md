```xml

    <param-value>0</param-value>
```

## 2. 🔴 핵심 버그: CHAR(8) 패딩으로 인한 조회 0건

### 증상
- 파라미터 정상 전달 확인 (478, 0005, 날짜 모두 OK)
- SQL 콘솔에 정상 출력
- DBeaver에서 동일 쿼리 실행 → 6건 조회
- Java `list.size()` → **0**

### 디버깅 과정 (삽질 목록)
1. resultType 경로 확인 → 정상
2. namespace, select id 확인 → 정상
3. XML 파일명 스캔 패턴 확인 → 정상
4. DB datasource URL/계정 확인 → 정상
5. Project Clean + Tomcat 재시작 → 미해결
6. 금리 컬럼 포맷 `#,##0.00` 발견 → String 매핑 실패 가능성 제기 (부분 기여)

### 진짜 원인
```
BIZ_CLSS_CD 컬럼 타입: CHAR(8)
DB 저장값:            '0005    '  (0005 + 공백 4개)
SQL 하드코딩 비교값:   '0005'      (4자리)
→ CHAR 패딩으로 인한 불일치 → WHERE 조건 불만족 → 0건
```

### 해결
- SQL 하드코딩 값을 CHAR(8) 자릿수에 맞게 수정

### 핵심 교훈
> **`CHAR(n)` 컬럼은 항상 n자리로 공백 패딩되어 저장된다.**  
> 쿼리에서 비교할 때 자릿수가 다르면 무조건 불일치 → 조회 안 됨  
> SQL 찍히고 DBeaver도 되는데 Java만 0건 → CHAR 패딩 의심

## 3. UNION ALL 컬럼 수 불일치 오류

### 에러
```
jdbc-8056: Invalid number of columns in query
```

### 원인
- UNION ALL 파트 간 SELECT 컬럼 수가 다름

### 해결
- 모든 UNION ALL 파트의 SELECT 컬럼 수 동일하게 맞추기
- 컬럼이 없는 파트는 `NULL AS 컬럼명` 으로 자리 채우기

---

## 4. 금리 컬럼 소수점 포맷 수정

- DB 저장값: 정수형 (예: `900000002`)
- 기존 그리드 format: `#,##0.00` → 소수점 2자리 표시
- 수정: `#,##0` → 소수점 제거
- **포맷은 프론트 XML format 속성에서 제어, 서버단 불필요**

## 5. WebSquare 그리드 헤더 동적 변경

### 잘못된 API (지원 안 함)
```js
grd_list.setColumnHeaderLabel("ctrtNo", headerNm); // ❌ not a function
```

### 올바른 API
```js
grd_list.setHeaderValue("columnId", "헤더명"); // ✅
```
- 주의: 컬럼 `id` 속성값 사용 (필드명 아님) → Design 탭에서 실제 id 확인 필수

### 구현 패턴 (DataMap onmodelchange 활용)
```xml
<!-- XML -->
<w2:dataMap id="dma_rclcInrtMngReqVO" onmodelchange="scwin.dma_rclcInrtMngReqVO_onmodelchange">
```

```js
// JS
scwin.dma_rclcInrtMngReqVO_onmodelchange = function(info) {
    if (info.key == "bizClssCd") {
        var cd = dma_rclcInrtMngReqVO.get("bizClssCd");
        var headerNm = "";
        if (cd == "00050000") {
            headerNm = "조달번호";
        } else if (cd == "00010000") {
            headerNm = "유가증권번호";
        } else if (cd == "00020000") {
            headerNm = "계약번호";
        }
        grd_list.setHeaderValue("column75", headerNm);
    }
};
```
