## 1. 화면설계서 공통 오류 패턴

### 입출력 항목

| 오류 유형 | 예시 | 수정 방향 |
|---|---|---|
| 일시 필드 길이 오기 | 최종작업일시 String/8 | String/17/YYYY-MM-DD HH:MM:SS |
| Y/N 플래그 길이 오기 | 취소여부 String/3 | String/1 |
| 이름 필드 길이 오기 | 부서장 String/3 | String/100 (~~Nm 패턴) |
| 환율 타입 오기 | 기준환율 String/8 | BigDecimal/12 |
| 날짜 타입 불일치 | 계약체결일자 Date | String/8/YYYY-MM-DD 로 통일 |
| No 순번 불연속 | 1,2,5,7 | 1,2,3,4,5,6,7 연속 기재 |

### 필드 길이 치트시트 (재확인)
```
~~Nm  (명)     : 100
~~Cd  (코드)   : 8
~~Ymd (일자)   : 8  / YYYY-MM-DD
~~Dt  (일시)   : 17 / YYYY-MM-DD HH:MM:SS
~~Amt (금액)   : BigDecimal, 19
이율            : BigDecimal, 12 (12,6)
환율            : BigDecimal, 12
Y/N 플래그      : 1
```

## 2. 처리흐름 공통 오류 패턴

| 오류 유형 | 내용 |
|---|---|
| Step 번호 중복 | 같은 번호 두 번 사용 — 순서대로 채번 필요 |
| 필수항목 미명시 | "필수 입력 항목을 체크한다" 만 쓰고 어떤 항목인지 미기재 |
| 버튼 누락 | 엑셀 다운로드, 수정, 등록 버튼 처리흐름 미작성 |
| Service ID 누락 | 조회/저장 API URI 미기재 |
| 결재 Step 누락 | 그룹웨어 전자결재 이관이더라도 비고 또는 별도 Step 명시 필요 |
| 속성 오기 | 결재정보(자동세팅) 항목을 E로 표기 → R이어야 함 |

### R vs E 기준
- **R (Read Only)**: 시스템/결재흐름에서 자동 세팅되는 값 (승인ID, 승인일시, 기안자 등)
- **E (Editable)**: 사용자가 직접 입력/수정하는 값

## 5. 지정일자 당해연도 Validation 구현

### 삽질 과정

**1차 시도 — EZZ0012 오용**
```javascript
com.win.message("EZZ0012", "당해년도만 입력 가능합니다.", ...);
// → "지정일자는 당해연도만 입력가능합니다. 은(는) 필수입력 항목입니다." 출력
// EZZ0012 = 필수입력 템플릿이라 뒤에 자동으로 문구 붙음 → 용도 다름
```

**2차 시도 — 타입 불일치**
```javascript
var thisYear = String(new Date().getFullYear()); // "2026"
if(dsgnYmd != thisYear) { ... }
// → dsgnYmd가 "2026" 이 아니라 "20261231" 형태로 들어와서 항상 true
```

**3차 시도 — substring 미적용**
→ `dsgnYmd.substring(0, 4)` 로 앞 4자리만 비교해야 함

**4차 시도 — focus() 무한루프**
```javascript
com.win._message(..., function() {
    boxName.focus(); // ← blur 이벤트 재트리거 → 무한루프
    boxName.setValue("");
});
```

### 최종 정답 코드
```javascript
var thisYear = String(new Date().getFullYear());

if(!com.util.isEmpty(dsgnYmd) && dsgnYmd.substring(0, 4) != thisYear) {
    com.win._message(gcm.MESSAGE_TYPE.INFO, "지정일자는 당해연도만 입력가능합니다.", function() {
        boxName.setValue(""); // focus() 제거 — 재트리거 방지
    });
    return;
}
```

### 핵심 포인트
- **알림 메시지**: `com.win._message(gcm.MESSAGE_TYPE.INFO, "메시지", callback)`
- **날짜 비교**: `substring(0,4)` 로 연도 4자리만 추출해서 비교
- **무한루프 방지**: callback 안에서 `focus()` 호출 금지

## TIL
- **일시 필드**: 길이 8은 날짜만, 일시는 반드시 17 (YYYY-MM-DD HH:MM:SS)
- **EZZ 코드**: 용도 확인 필수 — EZZ0012(필수입력), EZZ0054(날짜형식), EZZ0055(날짜대소비교)
- **단순 알림**: `com.win._message(gcm.MESSAGE_TYPE.INFO, ...)` 사용
- **날짜 연도 추출**: `substring(0,4)`
- **blur 콜백에서 focus() 금지**: 무한루프 원인
- **R vs E**: 자동세팅값(결재정보 등)은 반드시 R
- **처리흐름 버튼 누락**: 엑셀다운로드, 수정, 등록 빠지기 쉬움 — 화면 Layout과 1:1 대조 필수
