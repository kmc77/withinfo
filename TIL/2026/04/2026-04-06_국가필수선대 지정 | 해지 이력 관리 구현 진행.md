## 2. selectNtnGiveBrkd 에 이력 조회 태우기

별도 메서드를 만들지 않고, **기존 보조금 조회 메서드 안에 이력 조회를 함께 set** 하는 구조 채택.

```java
@Override
public NtnEsntlGrshListVO selectNtnGiveBrkd(NtnEsntlGrshVO vo) throws Exception {
    NtnEsntlGrshListVO result = new NtnEsntlGrshListVO();
    result.setNtnGiveBrkdListVO(ntnEsntlGrshMapper.selectNtnGiveBrkd(vo));
    result.setNtnEsntlDsgnHistListVO(ntnEsntlGrshMapper.selectNtnEsntlDsgnBrkd(vo));
    return result;
}
```

**이유**: 프론트에서 목록 행 클릭 시 `sbm_selectNtnGiveBrkd` Submission 하나만 호출하기 때문.
별도 Submission 추가 없이 ListVO 하나에 두 데이터를 담아 반환.

## 3. SQL 쿼리 (selectNtnEsntlDsgnBrkd)

```xml
<select id="selectNtnEsntlDsgnBrkd"
        parameterType="kobc.ngs.sf.ps.csbs.vo.NtnEsntlGrshVO"
        resultType="kobc.ngs.sf.ps.csbs.vo.NtnEsntlDsgnHistVO">
    SELECT /* SQL_ID: ...NtnEsntlGrshMapper.selectNtnEsntlDsgnBrkd */
           A.BIZ_YR
         , A.GIVE_DSPR
         , B.DSGN_YMD
         , B.RMV_YMD
         , B.FRST_REG_DT
         , B.FRST_REG_USER_ID
         , B.EXNT_CN
      FROM TPS_NTN_ESNTL_GRSH_SBSD_GIVE_B A
         , TPS_NTN_ESNTL_GRSH_H B
     WHERE A.NTN_ESNTL_GRSH_NO = B.NTN_ESNTL_GRSH_NO
     ORDER BY B.HIST_SN DESC
</select>
```

- 전체 조회 (선대번호 필터 없음) — 목록 전체 이력 표시 목적
- `BIZ_YR`, `GIVE_DSPR`는 SBSD_GIVE_B 테이블에서, 나머지는 H 테이블에서

## 6. saveValidation 지정일자 당해연도 체크

```javascript
var dsgnVal = ica_dsgnYmd.getValue();
if (!com.util.isEmpty(dsgnVal)) {
    var dsgnYr = dsgnVal.substring(0, 4); // "20260315" → "2026"
    var thisYear = String(new Date().getFullYear());
    if (dsgnYr !== thisYear) {
        com.win.message("IZZ0051", "지정일자는 당해연도만 입력가능합니다.");
        return false;
    }
}
```

- `calendarValueType="yearMonthDate"` 유지 (연월일 전체 입력)
- validation에서 앞 4자리만 잘라 연도 비교
- `getValue()` null 방어 처리 포함

## 7. sbx_dsgnYn onchange 이벤트

지정/해지 셀렉트박스 변경 시 해지일자 자동 처리:

```javascript
scwin.sbx_dsgnYn_onchange = function() {
    var selectedVal = sbx_dsgnYn.getValue();
    if (selectedVal == "N") {           // 해지 선택
        var thisYear = new Date().getFullYear();
        ica_rmvYmd.setValue(thisYear + "1231"); // 당해연도 12/31
    } else {                            // 지정 선택
        ica_rmvYmd.setValue("");        // 해지일자 초기화
    }
};
```

XML 속성 추가 필요:
```xml
ev:onchange="scwin.sbx_dsgnYn_onchange"
```

## TIL

- **Service 메서드 중첩 방지**: 별도 메서드를 추가하기 전에 기존 Submission 호출 구조를 먼저 확인할 것. 하나의 Submission으로 처리 가능하면 ServiceImpl 내에서 함께 set하는 구조가 더 깔끔함
- **Submission target 배열 바인딩**: WebSquare Submission의 target은 배열로 여러 DataCollection에 동시 바인딩 가능 — `[{"id":"dlt_A","key":"keyA"},{"id":"dlt_B","key":"keyB"}]`
- **VO 필드명은 반드시 설계 스펙 확인**: camelCase 변환 시 실수 잦음 (`FRST_REG_DT` → `frstRegDt`, `EXNT_CN` → `exntCn`)
- **getValue().substring()**: WebSquare inputCalendar에서 `yearMonthDate` 타입은 `YYYYMMDD` 형식 반환 → 앞 4자리 substring으로 연도 추출 가능. null 방어 처리 필수
