# 2026-04-03_국가필수선대 지정/해지 이력 관리 개발 착수

## 오늘 한 것
- 국가필수선대 지정/해지 이력 관리 기능 요구사항 파악
- 신규 이력 테이블 설계 확정
- 관련 파일 네이밍 및 VO 구조 결정
- 기존 소스(NtnEsntlGrsh 계열) 분석

---

## 1. 배경 및 문제
- 국가필수선대는 지정 → 해지 → 재지정 반복 발생
- 기존 테이블 키가 선대번호 하나뿐 → 수정 시 덮어쓰기
- 최신 정보만 남고 이력 추적 불가 → 이력 테이블 추가로 해결

---

## 2. 테이블 구조

| 테이블 | 설명 | 키 |
|--------|------|----|
| `TPS_NTN_ESNTL_GRSH` | 국가필수선대 정보 | 선대번호 |
| `TPS_NTN_ESNTL_GRSH_SBSD_GIVE` | 보조금 지급 내역 | 선대번호 + 차수 |
| `TPS_NTN_ESNTL_GRSH_H` | 이력 테이블 (신규) | 선대번호 + 이력일련번호 |

- `TPS_NTN_ESNTL_GRSH_H`는 INSERT ONLY — UPDATE 금지, 누적 방식

---

## 3. 개발 항목 3가지

**① 이력 INSERT 처리**
- 등록/수정 이벤트 시마다 `TPS_NTN_ESNTL_GRSH_H`에 INSERT
- DELETE/UPDATE 금지, 항상 신규 INSERT로 누적

**② 지정/해지 내역 그리드 추가 (화면 하단)**
- `TPS_NTN_ESNTL_GRSH_H` + `TPS_NTN_ESNTL_GRSH_SBSD_GIVE` 조인
- 사업연도는 보조금 지급 테이블에만 존재 → 조인 필수
- SQL id: `selectNtnEsntlDsgnBrkd`

**③ 버튼 이벤트 코딩**
- 저장 후 수정 모드로 전환되도록 처리
- 수정 모드에서 해지일자 변경 후 재등록 → 이력 테이블에 추가 INSERT

## TIL

- **이력 테이블 설계**: 단일 키 테이블은 덮어쓰기로 이력 추적 불가 → INSERT ONLY 이력 테이블로 해결
- **조인 쿼리 VO**: 두 테이블 조인 결과는 기존 VO에 맞는 필드가 없을 가능성 높음 → 별도 VO 생성이 일반적
