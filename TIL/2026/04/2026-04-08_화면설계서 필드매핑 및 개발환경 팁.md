## 1. 화면 필드 매핑 검증 — 누락 항목

기본정보 그리드의 화면 값과 소스 매핑 목록 비교 시 빠진 필드:

| 화면 필드 | 필드명 | 비고 |
|---|---|---|
| 기준연월 | `crtrYm` | 목록에 `crtrYr`(기준연도)만 있었음 |
| 등록상세 | `regCn` | 누락 |

> 화면에 보이는 필드를 소스 목록과 1:1 대조할 때, 유사한 이름(crtrYr vs crtrYm)에 속지 말 것

---

## 2. 적용환율 필드 길이

- 포맷: `#,###.##` (소수점 2자리)
- 타입: `BigDecimal`
- DB 길이: **(12, 2)**
- 근거: 환율은 정수부 최대 4~5자리 + 소수점 2자리면 충분, 화면 포맷과도 일치

---

## 3. Eclipse — txt 파일 Shift+Tab 안 될 때

plain text 에디터는 기본적으로 Shift+Tab outdent 미지원.

**해결: 파일 확장자를 `.xml`로 변경**
- Package Explorer에서 파일 우클릭 → **Rename** → `.txt` → `.xml`
- XML 에디터로 열리면 Tab / Shift+Tab 정상 동작

---

## 4. Mapper XML — 로컬 전용 스키마 접두사 처리

로컬 DB 권한 문제로 `OWNR_MNST_ADV.` 접두사를 임시로 붙여서 사용.

```xml
<!-- TODO: 배포 시 OWNR_MNST_ADV. 접두사 제거 -->
, OWNR_MNST_ADV.TPS_NTN_ESNTL_GRSH_H B
```

> **배포 전 반드시 제거** — 소스 배포 시 접두사 남아있으면 운영 DB에서 스키마 오류 발생

## TIL

- **적용환율 길이**: `(12,2)` 가 무난한 기본값
- **Eclipse txt → xml 변환**: Rename으로 확장자 바꾸는 게 제일 빠름
- **로컬 한정 SQL 수정**: 반드시 TODO 주석으로 배포 전 제거 대상 명시할 것
