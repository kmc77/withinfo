## [1] 조회 SQL 작성 — `TSM_INTG_FILE_I` LEFT JOIN

### 1차 시도 (ANSI + ROW_NUMBER)

```sql
LEFT JOIN (
    SELECT FILE_GRP_ID, FILE_OTXT_NM,
           ROW_NUMBER() OVER (PARTITION BY FILE_GRP_ID ORDER BY FILE_SN) AS RN
      FROM TSM_INTG_FILE_I
) F1 ON A.ACIN_RPPP_FILE_GRP_ID = F1.FILE_GRP_ID AND F1.RN = 1
```

**`ROW_NUMBER` 라인에서 "missing FROM keyword" (ORA-00923 호환)**

### 2차 시도 — `MIN(FILE_SN)` 서브쿼리로 대체

ROW_NUMBER 원인 파지 말고 `(+)` 버전이랑 동일 패턴으로 통일:

```sql
LEFT JOIN (
    SELECT X.FILE_GRP_ID, X.FILE_OTXT_NM
      FROM TSM_INTG_FILE_I X
     WHERE X.FILE_SN = (SELECT MIN(Y.FILE_SN)
                          FROM TSM_INTG_FILE_I Y
                         WHERE Y.FILE_GRP_ID = X.FILE_GRP_ID)
) F1 ON A.ACIN_RPPP_FILE_GRP_ID = F1.FILE_GRP_ID
```

→ **42000 에러 (SQL syntax error)**

## [2] 42000 원인 — 콤마 조인 + ANSI LEFT JOIN 혼용 금지

기존 쿼리가 콤마 조인 스타일(`FROM A, B WHERE A.x = B.y`)인데 ANSI `LEFT JOIN` 끼워 넣으니까 Tibero가 거부.

### 해결 — 콤마 조인 + `(+)` 통일

```sql
FROM TPS_LOAN_INT_SPRT_BIZ_B A,
     /* 기존 1:1 INNER 조인 테이블들 */
     TXX_CODE B,
     TXX_CODE C,
     /* 신규 인라인뷰 4개 */
     (SELECT X.FILE_GRP_ID, X.FILE_OTXT_NM
        FROM TSM_INTG_FILE_I X
       WHERE X.FILE_SN = (SELECT MIN(Y.FILE_SN)
                            FROM TSM_INTG_FILE_I Y
                           WHERE Y.FILE_GRP_ID = X.FILE_GRP_ID)
     ) F1,
     /* F2, F3, F4 동일 */
WHERE 1=1
  AND A.ACIN_RPPP_FILE_GRP_ID            = F1.FILE_GRP_ID(+)
  AND A.RMMD_FILE_GRP_ID                 = F2.FILE_GRP_ID(+)
  AND A.SPRT_TRGT_EVDC_PPRS_FILE_GRP_ID  = F3.FILE_GRP_ID(+)
  AND A.LOAN_NTSH_FILE_GRP_ID            = F4.FILE_GRP_ID(+)
  /* 기존 INNER 조인 조건 + 필터 조건 */
```

규칙:
- 1:1 INNER 조인 = `(+)` 없이
- 신규 OUTER = `(+)` 붙임
- 위치 무관. `AND`는 교환법칙.

### `(+)` 함정 — 같은 테이블 별칭에 OUTER/INNER 섞으면 강등

같은 인라인뷰/테이블 별칭(`F1` 등)에 `(+)` 붙은 조건과 안 붙은 조건이 섞이면 **그 테이블은 INNER 취급됨.**

```sql
/* 잘못된 예 */
AND A.ACIN_RPPP_FILE_GRP_ID = F1.FILE_GRP_ID(+)   -- OUTER로 의도
AND F1.FILE_OTXT_NM IS NOT NULL                    -- ← F1이 INNER로 강등됨
```

→ fileGrpId가 NULL인 행은 결과에서 사라짐. 파일 없는 행 안 보임.

## [3] 정책 전환 — 버튼 라벨 "파일등록" 고정

`fileGrpId`만 셀에 박혀도 보이는 건 **"파일등록"** 으로 통일이 정책.

### 함정

WebSquare 그리드 버튼 컬럼은 **cell data가 라벨 override함.** `value="파일등록"` 박아도 cell에 "20260530-0001" 들어오면 그게 표시.

### 해결 — 컬럼 분리

```xml
<!-- 데이터 보유 (hidden) -->
<w2:column id="acinRpppFileGrpId" visible="false"/>

<!-- 버튼 표시 (cell data 안 박음) -->
<w2:column id="acinRpppFileBtn" 
           inputType="button" 
           value="파일등록"
           displayMode="edit">
    <w2:onclick>scwin.btn_addFile_onclick</w2:onclick>
</w2:column>
```

클릭 핸들러는 컬럼 인덱스 비교만 `Btn` 컬럼 기준으로 변경. 데이터 fetch는 hidden 컬럼(`acinRpppFileGrpId`)에서:

```javascript
var idxAcinBtn = this.getColumnIndex("acinRpppFileBtn");
if(col == idxAcinBtn) { colId = "acinRpppFileGrpId"; ... }
var fileGrpId = dlt_loanIntSprtBizListVO.getCellData(row, colId);
```

**LEFT JOIN은 유지** — 파일명이 다른 용도(툴팁, 다운로드, hover 등)로 필요할 가능성 있음. 사수 컨펌 영역.

## [4] 엑셀다운 시 특정 컬럼만 제외하기 — 해결 ✅

`com.data.downloadGridViewExcel(grd, option, infoArr)` 함수에 **`removeColumns`** 옵션 존재.

- `excelExport` 속성은 이 프로젝트 사용처 0건 → 안 먹음
- `removeColumns`에 **컬럼 인덱스를 `","`로 이어붙여서** 전달

```javascript
var grd = grd_loanIntSprtBizListVO;

var removeIdx = [
    grd.getColumnIndex("acinRpppFileBtn"),
    grd.getColumnIndex("rmmdFileBtn"),
    grd.getColumnIndex("sprtTrgtEvdcPprsFileBtn"),
    grd.getColumnIndex("loanNtshFileBtn")
].join(",");

var option = {
    fileName: "대출이자지원사업",
    removeColumns: removeIdx
};

com.data.downloadGridViewExcel(grd, option, infoArr);
```

→ 화면 표시는 그대로, 엑셀에서만 제외.

## 핵심 교훈

| 구분 | 내용 |
|---|---|
| **Tibero 조인 혼용** | 콤마 조인(`FROM A, B`)과 ANSI `LEFT JOIN` 같은 쿼리에 섞으면 42000. 둘 중 하나로 통일. |
| **호환성 우선순위** | `ROW_NUMBER()` 분석함수 < `MIN()` 서브쿼리. 후자가 더 안전하고 `(+)` 버전과 동일 패턴. |
| **인라인뷰 별칭** | Oracle/Tibero는 `) AS F1` 금지. `) F1` 만 가능. (컬럼 별칭엔 `AS` OK) |
| **`FILE_SN`** | 그룹 내 순번이 아니라 전체 시퀀스. `FILE_SN = 1`로 못 잡음 → `MIN(FILE_SN)` 또는 `ROW_NUMBER` 필요. |
| **WebSquare 그리드 버튼 라벨** | cell data가 들어오면 `value` 속성 override함. 라벨 고정하려면 컬럼 분리 (hidden 데이터 + 표시 전용 버튼). |
| **WebSquare 컬럼 hidden 속성** | 표준은 **`visible="false"`**. `hidden`이라는 속성명은 없음. `displayMode`는 edit/read 제어용(별개). |
| **`excelExport` 속성** | 표준 옵션이긴 한데 이 프로젝트 코드베이스에 사용처 0. 안 먹음. |
| **공통 엑셀다운 컬럼 제외** | `com.data.downloadGridViewExcel` 옵션 중 **`removeColumns`** 사용. 컬럼 인덱스를 `","`로 이어붙여 문자열 전달. |
| **`WHERE 1=1`** | `AND` 콤마 관리 패턴. 신규 조건 추가 시 위치 무관, 그냥 맨 앞 유지하면 됨. |

## 회고

- **잘못 판단한 것 1**: 라벨 고정 정책 듣고 "LEFT JOIN 제거" 제안 → 파일명이 다른 용도로 필요할 수 있는데 단정함. **사용처 확인 없이 제거 제안하면 안 됨.**
- **잘못 판단한 것 2**: `WHERE 1=1`을 "기존 1:1 조인"으로 오독. 컨텍스트 확인 안 하고 답변.
- **잘 한 것**: ROW_NUMBER 에러 원인 깊게 파지 않고 `MIN` 서브쿼리로 우회 — `(+)` 버전과 동일 패턴이라 더 안전.
