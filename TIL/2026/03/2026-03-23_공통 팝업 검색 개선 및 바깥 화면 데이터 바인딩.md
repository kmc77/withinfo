> 날짜: 2026-03-23  
> 스택: WebSquare5 · Spring MVC · MyBatis · MariaDB

## 작업 1 — 공통 팝업 상세검색 구조 개선

### 문제

`sbx_choiceBox` (고객코드/이름/고객구분) + `sbx_TYPE` (일반/신혼부부/다자녀) + `ibx_searchKeyword` 세 개가 같은 `<td>` 안에 나란히 붙어 있어서:

- 고객구분 선택 시 sbx_TYPE이 갑자기 끼어져 보이는 혼란스러운 구조
- 어느 걸 입력해야 하는지 불명확

### 해결 방향

1. `sbx_choiceBox`에서 고객구분 항목 제거 → 고객코드/이름만 남김
2. 고객구분 셀렉트박스를 별도 `<tr>` 행으로 분리
3. `--선택--` → `--전체--` 로 변경해 전체 조회 기본값 처리
4. 보증상태 필터 `<tr>` 추가 (LOAN_APPLY JOIN 필요)

### 개선 전 구조

```
[sbx_choiceBox: 고객코드/이름/고객구분] [sbx_TYPE] [ibx_searchKeyword]
→ 모두 같은 td 안에 나란히 배치
```

### 개선 후 구조

```
[검색항목] [고객코드/이름 ▼] [키워드 input]
[고객구분] [전체/일반/신혼부부/다자녀 ▼]
[보증상태] [전체/심사중/심사거절/승인대기/승인완료/승인거절 ▼]
```

### XML 핵심 변경

```xml
<!-- 검색항목 tr: 고객코드/이름만 -->
<xf:select1 id="sbx_choiceBox" ref="data:dm_search.TYPE" style="width:150px;" appearance="minimal">
    <xf:choices>
        <xf:item><xf:label><![CDATA[고객코드]]></xf:label><xf:value><![CDATA[USER_ID]]></xf:value></xf:item>
        <xf:item><xf:label><![CDATA[이름]]></xf:label><xf:value><![CDATA[USER_NM]]></xf:value></xf:item>
    </xf:choices>
</xf:select1>

<!-- 고객구분 별도 tr -->
<xf:select1 id="sbx_TYPE" ref="data:dm_search.USER_TYPE_VAL" style="width:150px;" appearance="minimal">
    <xf:choices>
        <xf:item><xf:label><![CDATA[--전체--]]></xf:label><xf:value><![CDATA[ALL]]></xf:value></xf:item>
        ...
    </xf:choices>
</xf:select1>

<!-- 보증상태 별도 tr -->

    <xf:choices>
        <xf:item><xf:label><![CDATA[--전체--]]></xf:label><xf:value><![CDATA[ALL]]></xf:value></xf:item>
```

> **포인트**: label은 한글 표시용, value는 DB 실제 저장값 (01. 심사중 등) 으로 분리

### JS 수정 (btn_search_onclick)

```javascript
scwin.btn_search_onclick = function () {
    var choice = sbx_choiceBox.getValue();
    var searchWord = ibx_searchKeyword.getValue() || "";
    var userTypeVal = sbx_TYPE.getValue();
    var curStateVal = sbx_STATE.getValue();

    currentPage = 1;
    dm_search.set("TYPE", choice);
    dm_search.set("KEYWORD", searchWord);
    dm_search.set("USER_TYPE_VAL", userTypeVal);
    dm_search.set("CUR_STATE_VAL", curStateVal);
    dm_search.set("PAGE", currentPage);
    dm_search.set("PAGE_SIZE", 15);

    com.sbm.execute(sbm_searchUserInfoPopup);
};
```

### MyBatis 쿼리 개선

**개선 전**: USER_INFO 단독 조회

**개선 후**: LOAN_APPLY LEFT JOIN + DISTINCT + 서브쿼리로 페이징

```xml
SELECT (@rownum := @rownum + 1) AS SEQ, T.*
FROM (
    SELECT DISTINCT
        UI.USER_ID, UI.USER_NM, ...
    FROM USER_INFO UI
    LEFT JOIN LOAN_APPLY LA ON UI.USER_ID = LA.USER_ID
    <where>
        <if test="TYPE == 'USER_ID' and KEYWORD != null and KEYWORD != ''">
            AND UI.USER_ID LIKE concat('%',#{KEYWORD},'%')
        </if>
        <if test="USER_TYPE_VAL != null and USER_TYPE_VAL != '' and USER_TYPE_VAL != 'ALL'">
            AND UI.USER_TYPE = #{USER_TYPE_VAL}
        </if>
        <if test="CUR_STATE_VAL != null and CUR_STATE_VAL != '' and CUR_STATE_VAL != 'ALL'">
            AND LA.CUR_STATE = #{CUR_STATE_VAL}
        </if>
    </where>
    LIMIT #{PAGE_SIZE} OFFSET #{offset}
) T, (SELECT @rownum := #{offset}) r
```

### 트러블슈팅 기록

| 문제 | 원인 | 해결 |
|------|------|------|
| `--전체--` 선택 시 WHERE절에 `USER_TYPE = '--전체--'` 쿼리 날아감 | WebSquare selectbox value 빈값이 실제로 전달 안 됨 | value를 `ALL`로 명시하고 MyBatis에서 `!= 'ALL'` 조건 추가 |
| 고객구분 필터 적용 시 그리드 데이터 없음 | dm_search에서 값 가져오는 방식 오류 | `sbx_TYPE.getValue()` 직접 호출로 변경 |
| 페이징 이동 시 SEQ 1부터 다시 시작 | `@rownum := 0` 고정 | `@rownum := #{offset}` 으로 수정 |
| 보증상태 필터 결과 0건 | DB 저장값 `'01. 심사중'` 인데 `'심사중'` 으로 날아감 | label/value 분리 — label: 심사중, value: 01. 심사중 |
| DISTINCT 적용 후 한 고객이 여러 번 노출 | 한 고객의 신청 건수만큼 JOIN row 증가 | DISTINCT + 서브쿼리 구조로 변경 |

## 작업 2 — 팝업에서 바깥 화면으로 데이터 바인딩

### 구조

팝업(Common_Popup01)에서 행을 더블클릭하면 `closePopup(returnValue)` 로 값을 넘기고, 바깥 페이지의 콜백 함수에서 받아 바인딩한다.

### 팝업 내부 (더블클릭 핸들러)

```javascript
scwin.dlt_box_oncelldblclick = function(info) {
    let rowIdx = info;
    let rowData = dlt_User.getRowJSON(rowIdx);

    let returnValue = {
        USER_ID: rowData.USER_ID,
        USER_NM: rowData.USER_NM,
        USER_TYPE: rowData.USER_TYPE
    };
    com.win.closePopup(returnValue);
};
```

### 바깥 화면 (신청/심사/승인 페이지)

```javascript
// 돋보기 버튼 클릭 → 팝업 오픈
scwin.btn_openUserPopup_onclick = function() {
    var opt = {
        id: "Common_Popup01",
        popupName: "고객정보",
        width: "900", height: "800"
    };
    var data = { callbackFn: "scwin.userPopupCallback" };
    com.win.openPopup("/cm/common/Common_Popup01.xml", opt, data);
};

// 콜백 — 더블클릭 시 자동 호출
scwin.userPopupCallback = function(returnValue) {
    sbx_choiceBox.setValue("USER_ID");
    dm_search.set("TYPE", "USER_ID");
    dm_search.set("KEYWORD", returnValue.USER_ID);
    ibx_searchKeyword.setValue(returnValue.USER_ID);
};
```

### 바깥 화면 검색 영역 구성

```
[검색항목 ▼] [키워드 input] [🔍 돋보기(팝업)]
[고객구분  ▼]
[조회 버튼]
```

> **설계 원칙**: 보증상태 필터는 바깥 화면 자체가 신청/심사/승인으로 구분되므로 불필요

## 배운 점

- WebSquare selectbox에서 빈값(`""`) 처리가 불안정 → 명시적 `ALL` 값 사용이 안전
- `DISTINCT` + `@rownum` 조합 시 서브쿼리로 감싸야 순번이 올바르게 동작
- MyBatis `<where>` 태그 안에 `WHERE 1=1` 넣으면 중복 → `<where>` 자체가 WHERE 역할
- 팝업 바인딩은 `callbackFn` 패턴 — 팝업에서 `closePopup(값)`, 바깥에서 콜백 함수로 수신
