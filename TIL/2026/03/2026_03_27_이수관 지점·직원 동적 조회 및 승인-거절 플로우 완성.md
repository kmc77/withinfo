## 1. 수관 탭 승인/거절 플로우

### 흐름
```
신청중 → (수관자 승인) → 이관완료  /  (수관자 거절) → 거절
```

### searchPending 쿼리 수정
`FROM_EMP_NM`만 있던 쿼리에 `FROM_EMP_NO` 원본값 추가.  
백엔드 approve 처리 시 FROM 권한 비활성화에 필요.

```xml
SELECT 
    t.TRANSFER_NO,
    t.FROM_EMP_NO,
    (SELECT emp_nm FROM employee WHERE emp_no = t.from_emp_no) AS FROM_EMP_NM,
    ...
FROM EMP_AUTH_TRANSFER t
WHERE t.TO_EMP_NO = #{EMP_NO}
AND t.TRANSFER_STATUS = '신청중'
```

### dlt_pending_list에 TO_EMP_NO 추가
세션 대신 `dm_pending_emp.EMP_NO`를 execute 직전에 전 행에 세팅.

```javascript
var toEmpNo = dm_pending_emp.get("EMP_NO");
for (var i = 0; i < dlt_pending_list.getRowCount(); i++) {
    dlt_pending_list.setCellData(i, "TO_EMP_NO", toEmpNo);
}
com.sbm.execute(sbm_approve);
```

### 왜 dlt_approve_target 포기했나
- gridView 바인딩 없는 dataList는 JS에서 직접 접근 불가 (WebSquare 제약)
- `com.data.getDataCollection()`도 실패
- hidden gridView 방법도 복잡도 증가
- **결론**: submission ref를 `dlt_pending_list`로 바꾸고 백엔드에서 `CHK_YN2 == "1"` 필터링

### Controller
```java
// approve
List<Map> list = (List<Map>) param.get("dlt_pending_list");
for (Map item : list) {
    if (!"1".equals(item.get("CHK_YN2"))) continue;
    transfer.transfer_Auth(item);   // TO에 권한 INSERT
    transfer.deleteFromAuth(item);  // FROM 소프트딜리트 (IS_USE)
    item.put("TRANSFER_STATUS", "이관완료");
    transfer.updateStatus(item);
}

// reject - updateStatus만 호출
item.put("TRANSFER_STATUS", "거절");
transfer.updateStatus(item);
```

### updateStatus Mapper
```xml
<update id="updateStatus" parameterType="map">
    UPDATE EMP_AUTH_TRANSFER
    SET TRANSFER_STATUS = #{TRANSFER_STATUS}
    WHERE TRANSFER_NO = #{TRANSFER_NO}
</update>
```
approve/reject 둘 다 재사용.

## 2. 이관 컨트롤러 신청 단계 분리

기존 `/with/sample/transfer`가 권한 이동까지 한 번에 처리하고 있었음.  
이수관은 신청 → 승인 2단계 구조이므로 신청 시점엔 `addHistory`만.

```java
@RequestMapping("/with/sample/transfer")
public @ResponseBody Map<String, Object> transfer_Auth(@RequestBody Map<String, Object> param) {
    Map dmTransfer = (Map) param.get("dm_transfer");
    transfer.addHistory(dmTransfer); // TRANSFER_STATUS DEFAULT '신청중'
    ...
}
```

`TRANSFER_DT`는 ServiceImpl에서 세팅 중이라 컨트롤러 수정으로 충분.

## 3. 지점 선택 → 직원 동적 필터링

### 배경
이수관의 본래 목적: **폐합 지점 담당자 권한을 다른 직원에게 넘기는 것**  
→ 지점 선택 → 해당 지점 직원 필터링 → 이관/수관자 선택 구조 필요

### 추가한 dataList / dataMap
```xml
dlt_branch_list      <!-- BRANCH_CD, BRANCH_NM -->
dlt_from_emp_list    <!-- EMP_NO, EMP_NM -->
dlt_to_emp_list
dlt_pending_emp_list
dm_from_branch       <!-- BRANCH_CD -->
dm_to_branch
dm_pending_branch
```

### 추가한 submission
```
sbm_searchBranch          → dlt_branch_list
sbm_searchFromEmpList     → dlt_from_emp_list
sbm_searchToEmpList       → dlt_to_emp_list
sbm_searchPendingEmpList  → dlt_pending_emp_list
```

### w2:choices 동적 바인딩 안 됨 (WebSquare 제약)
`<w2:choices ref="data:dlt_branch_list" ...>` XML 선언만으로는 런타임에 자동 갱신 안됨.  
submitdone에서 `addItem()`으로 직접 채워야 함.

```javascript
scwin.sbm_searchBranch_submitdone = function() {
    for (var i = 0; i < dlt_branch_list.getRowCount(); i++) {
        var cd = dlt_branch_list.getCellData(i, "BRANCH_CD");
        var nm = dlt_branch_list.getCellData(i, "BRANCH_NM");
        sbx_from_branch.addItem(nm, cd);
        sbx_to_branch.addItem(nm, cd);
        sbx_pending_branch.addItem(nm, cd);
    }
};
```

직원 드롭다운도 동일 패턴:
```javascript
scwin.sbm_searchFromEmpList_submitdone = function() {
    sbx_from_employee.removeAll();
    sbx_from_employee.addItem('==담당자==', '==담당자==');
    for (var i = 0; i < dlt_from_emp_list.getRowCount(); i++) {
        sbx_from_employee.addItem(
            dlt_from_emp_list.getCellData(i, "EMP_NM"),
            dlt_from_emp_list.getCellData(i, "EMP_NO")
        );
    }
};
```

### 지점 onchange 통합 핸들러
```javascript
scwin.branch_onchange = function(type) {
    if (type === 'from') {
        dm_from_branch.set("BRANCH_CD", sbx_from_branch.getValue());
        com.sbm.execute(sbm_searchFromEmpList);
    } else if (type === 'to') {
        dm_to_branch.set("BRANCH_CD", sbx_to_branch.getValue());
        com.sbm.execute(sbm_searchToEmpList);
    } else if (type === 'pending') {
        dm_pending_branch.set("BRANCH_CD", sbx_pending_branch.getValue());
        com.sbm.execute(sbm_searchPendingEmpList);
    }
};
```

### 백엔드
```java
// searchBranch - BRANCH_INFO 전체
// searchEmpByBranch - EMPLOYEE WHERE BRANCH_CD = #{BRANCH_CD}
// 같은 URL 재사용, 파라미터 키만 다름
```

## 4. 그리드 셀 색상 displayExp로 교체

### 기존 문제
`setCellColor()`는 호출 시점에만 적용 → 그리드 re-render 시 날아감 → 최신 데이터에만 색상 적용되는 증상

### 해결
gridView column에 `displayExp` 속성 사용:

```xml
<w2:column id="TRANSFER_STATUS" inputType="text"
    displayExp="value=='거절'?'color:red':value=='이관완료'?'color:blue':''">
</w2:column>
```

렌더링마다 자동 평가되므로 re-render 후에도 유지됨.

## 핵심 정리

| 포인트 | 내용 |
|--------|------|
| gridView 미바인딩 dataList | JS에서 직접 접근 불가 → submission ref를 기존 dataList로 변경 |
| w2:choices 동적 바인딩 | XML 선언 후 submitdone에서 `addItem()`으로 직접 채워야 함 |
| 이수관 단계 분리 | 신청: `addHistory`만 / 승인: 권한 이동 + 상태 업데이트 |
| 셀 색상 | `setCellColor` → `displayExp` (re-render 안정성) |
| 소프트 딜리트 | FROM 권한 비활성화는 `IS_USE` 플래그 (금융권 표준) |
